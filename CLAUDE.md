## ディレクトリ構成

プロジェクト構成は以下です。

```text
./
├── .github/                # CI/CD設定
├── .claude/                # Agentルール・Skills
├── app/                    # Next.js App Router
│   ├── [lang]/             # i18n 動的ルートセグメント
│   │   └── page.tsx        # トップページ（ja/en 対応）
│   ├── globals.css         # グローバルスタイル
│   ├── layout.tsx          # ルートレイアウト
│   └── page.tsx            # デフォルトロケールへのリダイレクト
├── dictionaries/           # i18n 翻訳ファイル
│   ├── en.json             # 英語
│   └── ja.json             # 日本語
├── public/                 # 静的アセット
├── biome.json              # Biome 設定
├── get-dictionary.ts       # 翻訳辞書ローダー
├── i18n-config.ts          # ロケール設定
├── proxy.ts                # ロケール検出・リダイレクト
├── next.config.ts          # Next.js 設定
├── postcss.config.mjs      # PostCSS 設定
├── tsconfig.json           # TypeScript 設定
├── package.json            # 依存関係
├── pnpm-lock.yaml          # pnpm ロックファイル
└── README.md               # プロジェクト概要
```

## Skills

React/Next.jsプロジェクトのコードスタイルとベストプラクティスに関するガイドラインは `.claude/skills` にSkills が定義されているため参照してください。

以下のスキルが利用可能です。

| スキル名                | 概要                                            | 使用するタイミング                                                                                                        |
| ----------------------- | ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `react-best-practices`  | パフォーマンス最適化ガイドライン                | バンドルサイズ削減、再レンダリング修正、データフェッチング最適化、SSRパフォーマンス改善、コンポーネントリファクタリング時 |
| `react-composition`     | スケールするコンポジションパターン              | 多数のブールpropを持つコンポーネントのリファクタリング、柔軟なコンポーネントライブラリ構築、再利用可能なAPI設計時         |
| `web-design-guidelines` | Webインターフェースガイドラインへの準拠レビュー | UIレビュー、アクセシビリティチェック、デザイン監査、UXレビュー、ベストプラクティス確認時                                  |

## 開発ガイドライン

設計、実装の際には以下のガイドラインに従ってください。

- 設計、実装は `.claude/skills` にあるスキルを必要に応じて利用してください。
- UI表示のテキストはリテラルに実装はしないでください。必ず翻訳ファイルに格納するか、定数定義のファイルを作成して一元管理してください。
- コンポーネントは再利用性を意識して実装してください。
- コンポーネント作成時は `AtomicDesign` の考え方のもと粒度を意識してフォルダ分けしてください。

### 開発コマンド

```bash
pnpm dev          # 開発サーバー起動
pnpm build        # プロダクションビルド
pnpm start        # プロダクションサーバー起動
pnpm lint         # Biome lint チェック
pnpm lint:fix     # Biome lint 自動修正
pnpm format       # Biome フォーマット自動修正
pnpm check        # Biome lint + format を一括修正
pnpm typecheck    # TypeScript 型チェック
```
